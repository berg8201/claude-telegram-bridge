Quickstart

Requirements: Node.js 18+, claude CLI in PATH, codex CLI in PATH (for fallback/routing), and a Telegram bot token + chat ID.

1. Clone and install

git clone https://github.com/berg8201/claude-telegram-bridge.git
cd claude-telegram-bridge
npm install && npm link
npm run hooks:install

2. Configure env

mkdir -p ~/.config/bridge
cp .env.example ~/.config/bridge/.env
# Then edit ~/.config/bridge/.env and set:
# BOT_TOKEN=...
# CHAT_ID=...

3. Start Telegram mode

bridge telegram
# In Telegram, run:
/doctor

If you only want terminal mode first, run bridge normal instead.

FAQ

Is this for teams or solo developers?

Both. Most users start solo, then share the same workflow pattern across projects.

Does it replace Claude or Codex CLIs?

No. It orchestrates them and adds routing, memory, and Telegram control around your existing tools.

How does fallback work?

When enabled, limit/rate events trigger switch logic from primary provider to fallback provider.

Is chat data persisted?

Yes. Session history and summary are persisted in a session file across restarts.
